技術的なポイント

IDEA AgentはClaude CodeというAIツール上に、ルール（.mdファイル）やSkillを定義して動かしています。
この「.mdファイルで作り込んだAIの振る舞い」をそのまま使えるかどうかが、デプロイ方法を選ぶ最大の判断基準です。

要するに、「今ローカルPCで動いているものを、どこで24時間動かすか」という問題です。

Details

各選択肢の詳細

推奨（本番運用）

A. 社内PC（常時稼働機）+ Claude Code Remote

社内ネットワークにPCを1台設置（Windows / Mac どちらでもOK）。
今ローカルPCで動かしているIDEA Agentの環境をそのまま移設するイメージ。
外部（tldv）からの信号を受け取り、AIが自動で処理を実行。

メリット

  • 今作り込んだルール・Skillがそのまま動く
  • .mdファイルを編集するだけで機能追加・修正が簡単
  • エンジニア以外でもルール追記が可能
  • データが社内に留まる（セキュリティ面で有利）
  • 社内に余っているPCがあれば初期費用ゼロ

デメリット・リスク

  • ハードウェアの管理が必要（故障・停電対応）
  • 常時起動のため電源・ネットワークの確保が必要
  • ネットワーク設定が必要（社内NW担当と連携）
将来候補

B. Cloud Run + Agent SDK

Google Cloud上にプログラムを配置。
MTGの信号を受け取ったときだけ起動するサーバーレス方式。
ハードウェア不要、Google側が管理。

メリット

  • ハードウェア管理が不要
  • 使った分だけ課金（アイドル時はほぼ0円）
  • Google Cloudの高い安定性
  • 将来的にスケールしやすい

デメリット・リスク

  • 今の.mdルールをPythonコードに書き直す必要がある
  • 開発工数が大きい（フルリライト）
  • 変更のたびにエンジニアの作業が必要
PoC（まず試す用）

C. GAS + Claude API

Google Apps Script（GAS）でWebhookを受け取り、
Claude APIを呼んで処理する最小構成。
まず動くかどうかを確認するためのPoC（実証実験）向け。

メリット

  • 初期費用ゼロ、すぐ始められる
  • 開発が簡単（1〜2日で構築可能）
  • Google Workspaceの既存環境で動く
  • ChatWork通知・GitHub保存は十分対応可能

デメリット・リスク

  • 1回の実行が6分まで（長い文字起こしは分割が必要）
  • 複雑なエージェント処理には不向き
  • スケールには限界がある
代替案

D. クラウドVM（VPS）

AWS/GCPで仮想マシンを月額契約し、24時間起動しっぱなしにする。
その中にClaude Codeをインストールし、.mdルール・Skillを配置。
社内PCと同じ動かし方だが、物理的な機器が不要。

メリット

  • ハードウェア故障リスクなし
  • リモートアクセスが容易
  • 今の.mdルールがそのまま使える

デメリット・リスク

  • 月額費用が高め（月5,000〜15,000円）
  • 常時稼働のため無駄が出やすい
  • macOS VMは提供が限られる

Roadmap

推奨する進め方

いきなり本番環境を構築するのではなく、段階的に進めることでリスクを最小化します。

STEP 1（1〜2日）
GASでPoC: 最小構成で動作確認
tldv Webhook → GAS → GitHubに文字起こし保存 + ChatWork通知
費用ゼロで「自動処理が動く」ことを実証する。
ここで得られるもの: tldv Webhookの動作確認、処理フローの検証
STEP 2（1〜2週間）
GASにAI処理を追加
文字起こしをClaude APIに渡してタスク抽出・要約を自動化。
ChatWorkに「MTG要約 + ネクストアクション」を自動投稿。
ここで得られるもの: 実運用レベルの自動通知、チームからのフィードバック
STEP 3（要検討）
本番環境の選定・構築
STEP 2の結果を踏まえて、本番環境を決定。
GASで十分 → そのまま運用。より高度な処理が必要 → Mac Mini or Cloud Runへ移行。
判断基準: 処理の複雑さ、6分制限に引っかかるか、エージェント機能が必要か

結論: まずGASで小さく始め、必要に応じてスケールアップ

初期費用ゼロ・開発1〜2日で「MTGが終わったら自動通知」を実現できます。
その結果を見て、本格的なインフラ投資（Mac Mini等）を判断するのが最もリスクが低いアプローチです。

GASのPoC → チームの反応を見る → 本番環境を決める
